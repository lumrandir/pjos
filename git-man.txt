Настройка git на Windows.

1. Скачиваем msysGIT вот отсюда:
     http://msysgit.googlecode.com/files/Git-1.7.11-preview20120710.exe
2. Устанавливаем скачанное (тупо щёлкая Next->Next->... и ничего не меняя)
3. Скачиваем TortoiseGit:
     для 32-битных систем:
       http://tortoisegit.googlecode.com/files/TortoiseGit-1.7.12.0-32bit.msi
     для 64-битных систем:
       http://tortoisegit.googlecode.com/files/TortoiseGit-1.7.12.0-64bit.msi
4. Устанавливаем TortoiseGit (Next->Next->...) и перезапускаемся по запросу.
5. После перезагрузки нужно сгенерировать ключ, с которым вас будет пускать в репозитарий:
   5.1 Запускаем Puttygen (Все программы -> TortoiseGit -> Puttygen)
   5.2 Нажимаем Generate и яростно водим мышкой по столу, пока ключ не сгенерируется
   5.3 Нажимаем Save public key и сохраняем файл, например, с именем id_rsa.pub
   5.4 Нажимаем Save private key и сохраняем файл с именем id_rsa (если спросит, точно ли мы хотим сохранить файл без пароля, нажимаем "Да")
   5.5 Содержимое самого верхнего текстового поля в окне генератора копируем полностью и отправляем мне (выглядит оно примерно так:
     ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAIEA3pVkHh+tm+ecCHJa6DopMYVN08cESKCJZ/7
     sR1DuFYVi0FfCP4Nox5TadLViF3DYo+tAMcSgpciH7dnzj7XJM1WQpZnD9O9V9PBWCM3UDU
     RrMl0EazqfCWjdrPdQbgWgfLKFXXZ0XVarE/OV4SIitXn5khntW4cQDjYaG4xy15k= rsa-key-20120911
   ), это и есть ваш публичный ключ.
6. Теперь пришло время склонировать репо, репо само себя не склонирует:
   6.1 Заходим в какую-нибудь папку (куда хотим положить склонированное репо)
   6.2 Щёлкаем правой кнопкой по пустому месту -> Git Clone...
   6.3 Откроется окно, где в поле URL нужно вставить следующее:
     git@github.com:lumrandir/pjos.git
   6.4 Чекаем галочку Load Putty Key и указываем путь к файлу приватного ключа (того, который мы сохраняли вторым, с расширением .ppk)
   6.5 Нажимаем OK, начнётся клонирование (в процессе которого может вылезти окно, предлагающее нам добавить хост в список известных (нажимаем "Да"))
   6.6 Если в конце концов мы не видим синюю надпись Success, жалуемся мне, и я пытаюсь разобраться.
7. Теперь проверяем, разрешено ли вам писать в репозитарий:
   7.1 Заходим с только что склонированный репозитарий (папка pjos)
   7.2 Открываем README.md (текстовым редактором) и пишем туда что-нибудь вроде "Здесь был пёс"
   7.3 Сохраняем файлик и снова щёлкаем правой кнопкой по пустому месту
   7.4 В меню выбираем Git Commit -> "master"; при первом коммите нас попросят ввести своё имя/мэйл; соглашаемся и вводим
   7.5 Наконец откроется окно с полем Message (куда пишем какую-нибудь ерунду (можете привет мне передать)), после чего нажимаем ОК
   7.6 Когда коммит закончится, нажимаем Push и OK; начнётся процесс отправки ваших изменений в репо (и опять было бы здорово увидеть синюю надпись Success в конце)
